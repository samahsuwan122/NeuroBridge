Security & privacy

Protecting sensitive data by design

Security is a first-class requirement, enforced at the API layer.

JWT authentication

Access tokens with refresh tokens.

Role-based access

Patient, family, doctor, therapist, and admin scopes.

Password hashing

Stored only as bcrypt/Argon2 hashes — never plain text.

Audit logs

Login, profile updates, notes, reports, and AI review.

Scoped data access

Families see linked patients; doctors see assigned patients.

Secure file uploads

Validated type and size for memory-album media.

FAQ

Common questions

Does NeuroBridge AI diagnose any condition?

No. NeuroBridge AI is not a diagnostic medical system. It does not diagnose, predict, score, or treat any condition. Exercises measure game performance only.

What does the AI actually do?

The AI provides AI-assisted support: supportive activity recommendations and plain-language performance summaries. Every output is not a medical diagnosis and not a medical assessment, and requires review by a qualified doctor or therapist.

Does the AI make clinical decisions on its own?

No. AI suggestions stay pending until a doctor or therapist reviews them, and clinical judgment always stays with qualified professionals.

What is available today?

The patient mobile app is live with five cognitive exercises, a Memory Album, and progress analytics, on a multi-role architecture with RBAC, JWT, and audit logs, and ten language directions.

What is planned next?

The roadmap adds the Doctor Portal, Family Portal, Admin Dashboard, AI Core, Reports Engine, and cloud deployment.

How is sensitive data protected?

Through JWT authentication, role-based access control, password hashing, audit logging, and role-scoped data access.
